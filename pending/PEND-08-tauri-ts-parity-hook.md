# PEND-08 — `tauri.ts` ↔ `bindings.ts` parity pre-commit hook

## Problem

`src/lib/bindings.ts` (~101 commands) is auto-generated by tauri-specta from Rust. `src/lib/tauri.ts` (~96 hand-written wrappers) layers ergonomics on top: throw-on-error semantics via `unwrap()`, parameter coercion (e.g. `parentId ?? null`), JSDoc, camelCase shapes.

When a Rust command signature changes, `bindings.ts` regenerates automatically (and the existing `ts_bindings_up_to_date` pre-commit hook catches drift in `bindings.ts` itself). But `tauri.ts` is hand-edited, and **drift between `bindings.ts` and `tauri.ts` is unchecked**. A renamed Rust parameter can land with the wrapper still using the old name → runtime error in production.

The existing analogous hook is `scripts/check-tauri-mock-parity.mjs` (~144 lines), which catches a complementary drift class: bindings.ts ↔ `src/lib/tauri-mock/handlers.ts`. PEND-08 adds the symmetric check for the wrapper layer.

## Existing analogue: `check-tauri-mock-parity.mjs`

Pattern to mirror:

1. Parse `bindings.ts` with regex (no AST, no extra deps): extract command names from the `commands = { … }` object.
2. Parse the target file (handlers.ts in the existing hook; `tauri.ts` in the new hook): extract exported wrapper names.
3. Diff into three sets: `missingNew` (in bindings, not wrapped, not allowlisted), `allowlistStale` (allowlisted but no longer in bindings), `extra` (wrapped but not in bindings).
4. Exit non-zero on `missingNew`; warn on `allowlistStale` and `extra`.
5. Allowlist is hardcoded in the script (mock-parity uses a `KNOWN_UNMOCKED` set with one-line comments).

## Hook design

### Algorithm

1. Read `src/lib/bindings.ts` and `src/lib/tauri.ts`.
2. Extract command names from bindings via the regex `/^\s+([a-zA-Z][a-zA-Z0-9]*):\s*\(/gm` (matches the `commands` object shape).
3. Extract wrapper names from `tauri.ts` via `/^export\s+(?:async\s+)?function\s+([a-zA-Z][a-zA-Z0-9]*)\s*\(/gm`.
4. Load `KNOWN_UNWRAPPED` allowlist (inline in the script, like the mock-parity pattern).
5. Compute the three sets, report.
6. **Phase 1 (this task):** check command-name parity only. **Phase 2 (deferred):** parameter-name parity per command.

### Why phase-1-only initially

Parameter-shape parity requires either parsing parameter destructuring on the wrapper side (`{ blockType, content, parentId }`) or a more forgiving regex. The mock-parity script is name-only and has caught drift effectively for the mock layer. Start name-only here too; add parameter parity in a follow-up if real drift bugs show up.

### Allowlist

A small set of commands the project deliberately does not wrap. Reviewer verified the current state: **101 commands in bindings.ts, 96 wrappers in tauri.ts, 5 unwrapped.** The 5 are MCP / GCal status-and-setter commands consumed directly via `commands.*` because they don't benefit from ergonomic wrapping:

- `getMcpStatus` — MCP read-only status
- `getMcpSocketPath` — MCP read-only socket path
- `getMcpRwStatus` — MCP read-write status
- `getMcpRwSocketPath` — MCP read-write socket path
- `getGcalStatus` / `forceGcalResync` / `disconnectGcal` / `setGcalWindowDays` / `setGcalPrivacyMode` — GCal status + control surface

(Some of these may already be wrapped by the time the hook lands — verify on first run; the count and identities are a snapshot.)

Inline in the script as `const KNOWN_UNWRAPPED = new Set([ ... ])` with one-line comments per entry. JSON file is overkill at the expected sizes.

## Implementation

**New script: `scripts/check-tauri-bindings-parity.mjs`** (~150 lines, mirrors `check-tauri-mock-parity.mjs` size).

**`prek.toml` entry**, placed adjacent to the existing `tauri-mock-parity` block (around lines 111-120):

```toml
# Tauri bindings ↔ wrapper parity: ensures every `commands.*` export from
# `src/lib/bindings.ts` (auto-generated tauri-specta surface) has either a
# matching ergonomics wrapper in `src/lib/tauri.ts` or sits on an explicit
# allowlist. Catches the drift class where a Rust command signature changes,
# `bindings.ts` regenerates, but the hand-written wrapper is missed.
[[repos]]
repo = "local"
[[repos.hooks]]
id = "tauri-bindings-parity"
name = "tauri bindings ↔ wrapper parity"
entry = "node scripts/check-tauri-bindings-parity.mjs"
language = "system"
pass_filenames = false
files = "^src/lib/(bindings\\.ts|tauri\\.ts)$"
```

## Sketch (~30 lines, name-only)

```javascript
#!/usr/bin/env node
import fs from 'node:fs'
import path from 'node:path'
import { fileURLToPath } from 'node:url'

const ROOT = path.resolve(path.dirname(fileURLToPath(import.meta.url)), '..')
const bindingsSrc = fs.readFileSync(path.join(ROOT, 'src/lib/bindings.ts'), 'utf8')
const tauriSrc = fs.readFileSync(path.join(ROOT, 'src/lib/tauri.ts'), 'utf8')

const KNOWN_UNWRAPPED = new Set([
  // 'someInternalCommand', // reason here
])

const commands = new Set()
for (const m of bindingsSrc.matchAll(/^\s+([a-zA-Z][a-zA-Z0-9]*):\s*\(/gm)) {
  commands.add(m[1])
}
const wrappers = new Set()
for (const m of tauriSrc.matchAll(/^export\s+(?:async\s+)?function\s+([a-zA-Z][a-zA-Z0-9]*)\s*\(/gm)) {
  wrappers.add(m[1])
}

const missing = [...commands].filter(c => !wrappers.has(c) && !KNOWN_UNWRAPPED.has(c)).sort()
const allowlistStale = [...KNOWN_UNWRAPPED].filter(c => !commands.has(c)).sort()
const extra = [...wrappers].filter(c => !commands.has(c)).sort()

let exit = 0
if (missing.length) {
  console.error('ERROR: commands missing a tauri.ts wrapper (add to wrapper or allowlist):')
  for (const c of missing) console.error(`  - ${c}`)
  exit = 1
}
if (allowlistStale.length) {
  console.warn('WARN: allowlisted commands no longer present in bindings.ts (clean up):')
  for (const c of allowlistStale) console.warn(`  - ${c}`)
}
if (extra.length) {
  console.warn('WARN: tauri.ts has wrappers for commands not in bindings.ts (likely dead code):')
  for (const c of extra) console.warn(`  - ${c}`)
}
if (exit === 0) {
  console.log(`OK: ${wrappers.size}/${commands.size} commands wrapped.`)
}
process.exit(exit)
```

## Testing

```bash
prek run tauri-bindings-parity --all-files
```

Expected today: PASS (or a small number of legitimately-unwrapped items moved to the allowlist).

If it fails on first run, that's the discovery — it means current bindings.ts and tauri.ts already have drift. Either add the missing wrapper or move to the allowlist with a justifying comment.

## Impact

**Closes a real drift class.** The wrapper layer's value is ergonomics, but its hand-written nature is its weak point. Pre-commit catches the gap at the right time (commit, not CI, not user report).

Low ongoing maintenance: the regex is conservative; the allowlist is small; the hook runs in milliseconds.

## Risk

**Low.** Pre-commit only. Failures block the commit with a clear message and an obvious fix path. No production impact.

## Open questions

1. **Should the hook also validate parameter names?** Phase 1 says no — start with name parity. Add parameter-name parity in a follow-up only if real drift shows up after Phase 1 ships. (The mock-parity hook is name-only and has been sufficient.)
2. **Should the hook understand camelCase ↔ snake_case mapping?** Both files use camelCase consistently in their export surfaces, so no — direct string match works.
3. **Allowlist file vs inline?** Inline (consistent with mock-parity). If the allowlist grows past ~10 entries, switch to a JSON file.
