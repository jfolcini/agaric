# Agaric — Consolidated Analysis Report (2026-07-22)

All findings below survived adversarial verification against the real code; line citations were independently reproduced. Duplicates against open issues are marked. **No finding is superseded by the planned iroh transport swap (#78)** — each was explicitly checked; everything here lives in op-application semantics, projection, UI, tooling, or release paths, not the doomed transport layer.

## Executive summary

Agaric is in unusually good shape for a codebase of this size: the CRDT-as-source-of-truth architecture is coherent and consistently enforced, hot render paths are demonstrably state of the art (the dedicated perf audit returned a clean bill on virtualization, memoization, lazy-loading, and worker offload — gid 53), invariants are documented in-code with issue numbers, and the test/CI apparatus (conformance fixtures, mutation lanes, fuzz targets, parity guards) is beyond what most projects ship. The dominant risk theme is **semantic duplication**: op-log projection semantics live in five hand-synchronized interpreters (a class that has already shipped bugs #891/#1257), the entire e2e suite runs against a 5K-line JS reimplementation of the backend, and a 2,936-line hand-written IPC facade duplicates the generated bindings — each a drift machine. The second theme is **the local-first promise leaking at the edges**: exports silently drop block-scoped attachments, custom properties, and block refs; the updater hard-fails for deb/rpm users and cross-grades MSI installs; boot-fatal errors exit with zero user-visible surface. Third, several **correctness gaps in the move/lifecycle family** (nested-page page_id flattening, dual CRDT space membership, missing FK guard) are real but bounded, and fixes are well-scoped. Finally, the decomposition program (#2878) is validated by multiple independent findings that add concrete, unenumerated slices to it. Nothing found threatens data integrity at rest; the highest-severity single item is a remotely triggerable whole-process crash from the link-metadata HTML parser.

## Architecture — backend

- **[HIGH] 5 parallel op-log interpreters** (gid 0, new) — `apply_op_tx` (kernel.rs:332), nine sql_only fallbacks, two independent `match op_type` replay loops (recovery.rs:603, :1073), and crud.rs's hand-mirrored delete arm (:753 "mirrors apply_op_tx's DeleteBlock arm exactly") are synchronized manually; this class already produced #891 and #1257. Converge recovery onto `apply_op_projected`/projection fns (edit_block already is, recovery.rs:687) and extract a shared lifecycle-apply helper. Care: some recovery divergence is deliberate (#618 era encoding, #1536 idempotent replay). Effort L.
- **[MED] Reverse/undo kernel stranded in the IPC layer** (gids 1+41 merged — two analysts independently; overlaps #2878 item 5, delta is new) — the forward kernel moved to agaric-engine (#2621) but `apply_reverse_in_tx` (history.rs:444, a 427-line 11-arm match), `drive_reverse_engine` (:376) and friends sit in the 3,121-line highest-churn-quartile `commands/history.rs`, with inverse-op *computation* in a third place (`src/reverse/`). - **[MED] ~26K LOC of sync tests stranded in agaric_lib behind namespace-reconstruction shims** (gid 4; overlaps #2878 item 4, delta new) — sync_daemon/tests.rs (9,413 lines behind a 33-line shim) etc. force sync internals `pub` and narrow coverage to the public surface. - **[LOW] Boot-replay suppression is an ambient global AtomicBool** (gid 5, new) — `replay_suppress_reproject` (shared.rs:74) soundness rests on a comment-enforced quiescence invariant ("do NOT relax"). ## Robustness — backend

- **[MED] Local attachment writes are non-atomic and unfsynced** (gid 28, new) — plain `std::fs::write` (sync_files.rs:675) before the WAL-fsynced row commit; post-commit power loss = durable row, truncated bytes; no read-path hash verification, and content-hash dedup makes a corrupt canonical blob self-perpetuating. The repo has the correct pattern twice (streaming receiver, device.rs). Mirror it: tmp + `sync_all` + rename + parent-dir fsync; optionally verify content_hash on read. Effort S.
- **[MED] Boot-fatal failures exit silently; no pre-migration backup** (gids 29+88 merged — two analysts converged, new) — any setup error (corrupt page, failed `sqlx::migrate!`, engine reprojection, and specifically `MigrateError::VersionMissing` on downgrade-after-failed-update, which sqlx fails closed on) hits `exit(1)` with only a log line (lib.rs:2338); no dialog plugin exists; migrations run against the only copy of notes.db. Show a native message box before exit (targeted message for VersionMissing: "opened by a newer version — reinstall latest"), copy notes.db+wal to a timestamped backup before migrating at a version boundary. Effort S–M.
- **[MED] Engine-first corruption rebuild is all-or-nothing with a one-shot gate** (gid 30, adjusted, new) — per-space decode failures skip+continue but any per-block error `?`-propagates out of a single shared tx (recovery.rs:411-436), failing that boot; the gate then never re-fires, so the outcome is one failed boot followed by permanent, silent loss of the entire engine-first recovery (not a boot brick). Wrap each space in a SAVEPOINT, log+continue, matching the module's own philosophy. Effort M.

## Deeplink / outbound-URL security

- **[HIGH] Remote whole-app crash via Unicode-lowercase offset desync in the HTML parser** (gid 74, new) — three functions compute offsets on `to_lowercase()` (which changes byte length: İ, ẞ, Kelvin K) and slice the original string; hand-traced PoC: a page body `İ<title>中x</title>` panics mid-char, and `panic="abort"` (Cargo.toml:431, zero catch_unwind) kills the process; fetch fires automatically on link hover and paste. - **[MED] Link-preview favicon bypasses the external-image consent policy and SSRF guard** (gid 75, new) — `<img src={metadata.favicon_url}>` renders an attacker-controlled URL from fetched HTML with no `shouldLoadExternalImage` gate; CSP `img-src https:` permits any host, and the Rust SsrfGuardResolver never sees it — an uncontrolled hover-triggered beacon that fully bypasses the click-consent default. - **[LOW] Context-menu/long-press "Open link" lacks the isAllowedUrl layer** (gid 76, adjusted, new) — not currently exploitable (TipTap's renderHTML sanitizes hrefs; the static renderer emits no .external-link for blocked schemes), but it's the one open sink without the click sinks' defense-in-depth.
